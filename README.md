# Jenkins + k6 + Python Report Generator

Этот проект содержит CI/CD-пайплайн для Jenkins, который запускает нагрузочные тесты на **k6**, а затем выполняет Python-приложение для генерации итогового отчета.

## Что делает проект

Пайплайн автоматизирует полный сценарий прогона тестов:

1. Jenkins запускает k6-тест.
2. После завершения теста сохраняются артефакты или результаты выполнения.
3. Python-приложение собирает результаты тестирования и публикует.
4. На выходе формируется отчет в Confluence.

Такой подход позволяет отделить выполнение теста от логики подготовки отчетности и упрощает поддержку процесса анализа результатов.

## Логика пайплайна

Общий сценарий работы Jenkins pipeline:

- checkout исходного кода;
- запуск k6-теста;
- сохранение результатов теста в influxdb;
- запуск Python-скрипта, который собирает данные по прошедшему тесту;
- публикация готового отчета в Confluence.
